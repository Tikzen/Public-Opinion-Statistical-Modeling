# Public Opinion Statistical Modeling  
## 舆情传播统计建模与智能分析平台

---

## 1. 项目概述

**Public Opinion Statistical Modeling** 是一个面向舆情传播研究与教学展示的综合型统计建模平台。项目围绕“**真实数据驱动 + 参数反推 + 传播仿真 + AI分析**”这一主线展开，试图解决一个很实际的问题：

> 当我们观察到某一热点事件、谣言话题或公众议题在网络中的热度变化曲线后，能否反推出其背后的传播强度与衰减规律，并进一步在人工构造的社交网络上模拟其扩散过程，验证不同治理策略的效果？

为实现这一目标，本项目将以下几类方法整合为一个统一系统：

- **复杂网络建模**：使用无标度网络刻画社交网络结构；
- **传播动力学建模**：使用 SIR / SI / SIS 等传播模型描述扩散行为；
- **参数估计**：根据真实时间序列数据反向估计传播率 β 与恢复率 γ；
- **AI智能分析**：使用大语言模型对时间序列特征进行解释，并生成参数建议；
- **交互式可视化**：通过 Streamlit 提供完整的实验界面、图表展示与结果导出能力。

因此，这个项目并不只是一个“谣言传播模拟器”，而是一个具备**数据分析、模型估计、策略验证、结果解释**完整闭环的统计建模系统。

---

## 2. 项目目标与研究思路

本项目的核心目标有三个：

### 2.1 用真实数据驱动建模，而不是只做纯理论模拟
很多传播模型项目停留在“给定参数直接模拟”的层面，虽然能展示曲线，但很难说明模型与现实数据之间的联系。本项目引入了真实时间序列数据（如百度指数），先对传播热度进行提取与摘要，再反向估计模型参数，使系统具备“**从现实到模型**”的能力。

### 2.2 将参数估计与传播模拟连接起来
传统仿真系统与参数估计往往彼此割裂：一个模块负责生成曲线，一个模块负责拟合数据，而两者之间缺乏联动。本项目将“参数估计页”与“传播模拟页”连接起来，使拟合得到的参数可以直接用于后续模拟验证，从而形成“**拟合—仿真—比较**”的闭环。

### 2.3 引入 AI 模块提升可解释性与交互性
仅有参数数值并不足以支撑完整分析，尤其是在比赛展示中，评委更关心系统是否能够“解释结果”。因此本项目增加了 AI Analysis 模块，通过时间序列摘要、规则参数建议与大模型分析相结合，输出**中文参数建议、建模策略说明、传播特征解释**，提高系统的智能化程度与展示效果。

## 3. 系统整体架构

本项目采用分层设计思路，整体可以概括为以下结构：

```text
数据层 → 特征摘要层 → 参数建议层 → 参数估计层 → 传播仿真层 → AI解释层 → 可视化与导出层
```

### 3.1 数据层
负责读取外部数据。目前主要支持：
- 百度指数导出的 Excel / CSV 文件；
- 手动输入的时间序列数据；
- 自检模式下由模型自动生成的测试数据。

这一步的意义在于：系统既能支持现实数据，也能支持模型自生成数据，从而兼顾**真实应用**与**算法验证**。

### 3.2 特征摘要层
对输入时间序列进行统计特征提取，包括：
- 起始值、终止值；
- 最大值、最小值、均值、标准差；
- 峰值位置、峰值日期、峰值强度；
- 趋势判断（如先上升后下降）；
- 波动等级判断。

该层的作用是把原始序列压缩成更适合建模与 AI 理解的结构化信息，避免将整段原始数据直接送入大模型。

### 3.3 参数建议层
根据时间序列摘要，使用规则系统给出初步参数建议，包括：
- β 初值与搜索区间；
- γ 初值与搜索区间；
- 是否建议使用归一化拟合；
- 建议代理网络节点数；
- 建议每个新节点连接边数；
- 建议初始传播源类型；
- 建议随机种子平均次数。

这一层的思想是：**先由可控规则系统生成建议，再由 AI 负责解释与补充**。这样既保持了参数建议的稳定性，也避免大模型“自由发挥”导致不稳定。

### 3.4 参数估计层
这一层基于真实数据，通过多种优化方法寻找最优 β / γ，使模型生成的传播曲线尽量逼近真实序列。项目支持：
- 网格搜索；
- 两阶段搜索；
- 贝叶斯优化。

其中，“两阶段搜索”兼顾速度与精度；“贝叶斯优化”则进一步提升了搜索效率。

### 3.5 传播仿真层
在给定模型与参数后，在复杂网络上执行传播模拟。支持：
- 多模型切换；
- 不同传播源类型；
- 不同治理策略；
- 网络动态展示；
- 多轮传播曲线统计。

### 3.6 AI解释层
负责基于结构化摘要与参数建议，调用大语言模型输出中文分析结论，用于增强系统可解释性。AI 不直接代替算法，而是作为算法分析结果的“解释器”和“建议器”。

### 3.7 可视化与导出层
负责：
- 传播曲线展示；
- 网络传播动态可视化；
- 拟合结果图表；
- 自动实验报告；
- CSV / ZIP 导出。

## 4. 推荐使用流程

本系统最推荐的使用流程如下：

### 第一步：获取数据
用户先获取舆情传播相关时间序列数据。当前最适合的方式是使用百度指数等平台导出的搜索热度数据，因为这类数据天然具有“热度上升—峰值—回落”的传播特征，适合与传播模型进行对接。

### 第二步：进入 AI Analysis 页面
将数据文件上传到 AI Analysis 页面后，系统会：
1. 解析数据；
2. 读取时间序列；
3. 自动提取趋势、峰值、波动性等统计特征；
4. 生成中文参数建议；
5. 调用大模型输出中文分析文本。

在这一页面中，用户不仅可以看到 AI 的解释性结论，还可以得到一组更适合后续拟合的参数建议。

### 第三步：将参数同步到 Parameter Estimation 页面
AI Analysis 页支持一键同步，将建议的：
- β / γ 搜索范围；
- 代理网络节点数；
- 节点连接边数；
- 传播源类型；
- 随机种子数；
- 归一化拟合建议；

同步到参数估计页面左侧设置区，从而减少人工试错成本。

### 第四步：进入 Parameter Estimation 页面进行参数拟合
在参数估计页面中，系统会根据真实时间序列，通过不同优化方法估计最优传播参数。用户可以对比原始值拟合与归一化拟合效果，并查看：
- SSE / MSE / RMSE / MAE / MAPE / R²；
- 峰值误差；
- 峰值轮次误差；
- 拟合结论文本。

### 第五步：进入 Simulation 页面进行传播仿真与策略验证
将拟合结果用于传播模拟，在网络中观察舆情扩散过程，并验证：
- 无干预场景下的传播演化；
- 辟谣干预是否有效；
- 关键节点限制是否更优；
- 不同传播源类型的影响。

### 第六步：导出结果
系统可导出：
- 拟合结果 CSV；
- 仿真结果 ZIP；
- 自动生成的实验报告。

这使项目不仅适合系统演示，也方便用于论文写作、答辩展示和比赛提交。

## 5. 核心页面与功能说明

## 5.1 AI Analysis：AI智能分析页面

该页面是本项目中最具创新性的模块之一，用于将“真实数据分析”与“参数估计”连接起来。

### 主要功能

#### （1）数据读取与预览
支持上传百度指数 Excel / CSV 文件，并自动读取其中的时间序列数据。系统提供数据长度、关键词、前若干日期与数值预览，用于验证输入是否正确。

#### （2）时间序列摘要
页面可调用摘要模块，对原始数据进行统计分析，提取：
- 长度；
- 起止值；
- 最大值、最小值、均值、标准差；
- 峰值位置、峰值日期、峰值倍数；
- 趋势特征；
- 波动性等级。

这一过程相当于在 AI 调用之前，先由本地算法完成一层“统计建模预处理”。

#### （3）中文参数建议
系统会基于摘要结果，给出非常具体的中文参数建议，例如：
- 建议 β / γ 初值；
- 建议 β / γ 搜索上下界；
- 建议代理网络节点数；
- 建议每个新节点连接边数；
- 建议传播源类型；
- 建议随机种子平均次数；
- 建议拟合模式（原始值 or 归一化）。

这部分不是简单列名词，而是会生成带解释的中文建议文本，便于用户直接理解。

#### （4）LLM 智能分析
页面支持调用大语言模型进行进一步解释。当前配置支持：
- DeepSeek（优先）；
- 硅基流动（作为回退平台）。

当首选平台请求失败时，系统可自动切换备用平台，从而提高整体稳定性。AI 输出的内容主要围绕：
- 趋势分析；
- 峰值解释；
- 波动性说明；
- 参数建议合理性；
- 后续拟合策略。

#### （5）一键同步到参数估计模块
这是整个项目闭环设计的关键之一。AI Analysis 页生成的参数建议可以直接写入 `st.session_state`，在 Parameter Estimation 页面中自动读取，从而避免用户手动抄写参数。

#### （6）可复制数据（空格分隔）
系统还会自动把数值序列转换为“空格分隔”的格式，便于直接复制到参数估计页面的“手动输入”框中。这是一个非常实用的小功能，提升了跨页面使用效率。

## 5.2 Parameter Estimation：参数估计页面

该页面负责根据真实时间序列，反推传播模型参数。

### 主要功能

#### （1）支持多传播模型
页面与 `model/` 文件夹中的模型注册系统联动，支持：
- SIR；
- SI；
- SIS；
- 后续扩展模型。

每个模型通过统一接口加载，因此具备良好的扩展性。

#### （2）支持拟合目标状态切换
系统允许选择拟合目标状态。例如，对于 SIR 模型，通常拟合 `I`（感染状态）最有意义；若模型结构不同，也可切换其他状态。

#### （3）支持多种数据输入方式
页面支持三种输入方式：
- 手动输入时间序列；
- 上传 CSV 文件；
- 使用自检数据。

这使得页面既能处理真实数据，也能处理算法验证场景。

#### （4）自检模式
这是本页面非常重要的亮点。系统可以先用已知 β / γ 生成一组模拟数据，再反向拟合回去，检验算法是否能找回接近的参数。  
这一步的意义在于：先验证“参数估计方法本身是可靠的”，再处理真实数据。

#### （5）多种优化算法

##### ① 网格搜索
对 β / γ 在指定区间内做穷举搜索。优点是稳定、直观，缺点是速度较慢。

##### ② 两阶段搜索（推荐）
先在较粗的范围上进行低分辨率搜索，锁定大致最优区域；再在局部区域内进行细化搜索。  
该方法兼顾速度与精度，尤其适合比赛演示与常规实验。

##### ③ 贝叶斯优化
本项目引入 `scikit-optimize` 模块实现贝叶斯优化。  
其核心思想是：不再暴力遍历参数空间，而是基于高斯过程（Gaussian Process）对目标函数进行代理建模，根据当前搜索结果智能推断哪些区域更有可能出现更优参数，从而在更少的迭代次数下逼近最优解。

相比传统网格搜索，贝叶斯优化的优势在于：
- 所需评估次数更少；
- 搜索效率更高；
- 对连续参数空间更友好；
- 在计算开销较大时优势更明显。

这也是本项目技术亮点之一。

#### （6）多随机种子平均
由于网络传播仿真本身带有随机性，如果只用单个随机种子，拟合结果可能受偶然性影响。  
因此本项目支持“多随机种子平均”，即在多个种子下运行仿真，再对曲线取平均，从而降低拟合过程中的随机波动，提高结果稳定性。

#### （7）归一化拟合与原始值拟合
系统支持两种拟合模式：

- **原始值拟合**：直接拟合原始数量级，更适合关注绝对规模；
- **归一化拟合**：先将真实序列与模拟序列归一化，再比较趋势形状，更适合关注传播节奏与峰值位置。

在处理真实舆情数据时，由于现实数据常常存在数量级偏差、峰值极高或噪声较大等问题，归一化拟合往往更能稳定地反映传播趋势。

#### （8）结果展示
拟合完成后，页面会展示：
- 最优 β / γ；
- 各类误差指标；
- 原始值对比图；
- 归一化对比图；
- 误差表；
- 自动结论文本；
- 自检对照结果。

## 5.3 Simulation：传播模拟页面

该页面负责将传播模型应用到社交网络中，观察扩散过程并进行策略验证。

### 主要功能

#### （1）复杂网络构建
页面基于 BA 无标度网络生成代理社交网络。该网络特征是：
- 少数节点连接度极高；
- 大多数节点连接度较低；
- 更贴近现实社交网络中的“关键意见节点”现象。

#### （2）支持多传播模型
与参数估计页面相同，传播模拟页同样支持 SIR / SI / SIS 等模型，并通过统一模型接口实现扩展。

#### （3）支持不同传播源类型
初始传播源支持：
- 随机节点；
- 普通节点；
- 关键节点。

这一设计用于研究“传播源身份”对扩散规模的影响。例如，从关键节点发起传播，通常更容易形成大规模扩散。

#### （4）支持两类干预策略

##### 辟谣机制
系统可在指定轮次后降低传播概率，相当于在传播过程中引入官方辟谣、信息澄清或平台干预。

##### 关键节点限制
系统可限制高中心性节点的传播能力，用于模拟“封禁大号”“限制关键传播者扩散”等治理策略。

#### （5）策略对比实验
页面可同时运行：
- 无干预；
- 辟谣干预；
- 关键节点限制；

并将三条传播曲线放在同一图中进行比较。这一点非常适合答辩展示，因为它不仅展示了模型，还展示了“策略效果分析”。

#### （6）传播源对比实验
系统还支持比较不同传播源类型（随机 / 普通 / 关键）的扩散曲线与峰值差异，用于分析传播起点对最终结果的影响。

#### （7）网络动态演化展示
页面支持逐轮展示网络节点状态，并可播放传播动画。新增感染节点会被高亮显示，使传播过程具有较强的可视化表现力。

#### （8）自动实验报告与结果导出
页面会根据本次实验参数与结果，自动生成实验报告文本，并支持：
- 导出实验报告；
- 导出 ZIP 压缩包；
- 导出图表与结果数据。

这使页面不仅具备演示功能，也具备实验记录能力。

## 6. 技术亮点与创新点

### 6.1 模块化模型加载机制
项目通过 `pkgutil` 与 `importlib` 实现模型动态加载，使 `model/` 文件夹中的传播模型可以自动注册进入系统。  
这一设计避免了每增加一个模型就要手动改 UI 和主逻辑，提高了扩展性。

### 6.2 参数估计与传播模拟闭环
很多项目把“拟合”和“仿真”分离开来，而本项目把两者联动起来：AI Analysis 给建议，Parameter Estimation 做反推，Simulation 做验证，形成完整闭环。

### 6.3 AI 驱动参数推荐
本项目的 AI 模块不是简单生成一段说明文字，而是能够：
- 理解输入数据；
- 提取趋势；
- 生成参数建议；
- 同步到拟合模块。

这一点使 AI 真正参与建模流程，而不是仅作为展示附属品。

### 6.4 贝叶斯优化引入
引入 `scikit-optimize` 实现高斯过程驱动的贝叶斯优化，显著降低参数搜索所需时间，提高高维连续参数搜索效率。

### 6.5 多随机种子平均机制
通过多随机种子仿真平均曲线，减少单次传播模拟的偶然性，提高拟合稳定性。这种设计非常适合传播模型这类高随机性问题。

### 6.6 自检机制
自检模式确保参数估计不是“黑箱输出”，而是可以通过模型自生成数据验证其有效性。这在比赛答辩中是非常有说服力的设计。

### 6.7 中文参数建议与交互优化
系统不仅提供结构化参数建议，还提供完整中文解释与跨页面同步能力，大幅改善用户体验。

## 11. 项目总结

Public Opinion Statistical Modeling 并不是单一的传播模拟器，而是一个围绕舆情扩散问题构建的**综合型统计建模平台**。  
它将真实数据分析、参数估计、复杂网络仿真、AI解释与可视化展示结合起来，形成了一条完整的分析链路：

> **真实数据 → 特征摘要 → 参数建议 → 参数拟合 → 传播仿真 → 策略验证**

这一闭环结构使项目兼具：
- 学术建模价值；
- 工程实现价值；
- 交互展示价值；
- 竞赛答辩价值。

从比赛角度看，本项目的优势不只在于“功能多”，更在于模块之间的逻辑连接清晰，能够体现完整的问题分析思路与系统实现能力。
